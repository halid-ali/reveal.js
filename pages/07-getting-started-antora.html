Getting Started with Antora

Install Antora

$ npm i -D -E @antora/cli@3.1 @antora/site-generator@3.1
$ npx antora -v

The first step is to install Antora on your system. You can do this using a package manager like npm. After the installation, you can verify the installation by checking version.

click for next

Once Antora is installed, you'll need to create a directory for your documentation. This directory should contain your documentation files in AsciiDoc format, along with an Antora playbook file.

click for next

The playbook file is a YAML file that contains configuration options for Antora. You'll need to configure the playbook to point to your documentation files. We will explore the concept of a playbook file shortly.

click for next

Once your playbook is configured, you can generate the site using the antora command.

Antora Playbook File
site:
  title: My Documentation
  url: https://my-docs.example.com
  start_page: index.adoc
content:
  sources:
  - url: https://github.com/my-org/my-docs
    branches: master
    start_path: docs
output:
  directory: public
  format: site

Here is a sample playbook file that contains fundamental properties.

This playbook tells Antora to generate a site with the title "My Documentation", using documentation files located in the docs directory of the my-org/my-docs repository on GitHub. The generated site will be placed in the public directory

Once you've generated the site, you can view it in a web browser to see how it looks. If you need to make changes to your documentation, you can edit the AsciiDoc files and re-run the antora command to regenerate the site.
