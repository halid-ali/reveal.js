Antora collects and processes content source files according to a set of hierarchical files and directories it expects to find at a content source root.


  1. Content source root.
  2. The descriptor file which is required at each content root
  3. It must be located at the same level as antora.yml file
  4. ROOT is reserved directory name and must be in all uppercase
  5. Pages family directory (reserved directory name)
  6. Partials family directory (reserved directory name)
  7. Navigation file
  8. It won’t be processed because located outside the modules

Antora has its own specific file and directory set to collect and process the content.

At very top, we have root folder for the content source which contains the descriptor file, modules and packages

The descriptor file must be located in the content root. This file indicates to Antora that the modules directory should be collected and processed. We will have a look at it later on.

Then the modules directory which consists our content and must be located at the same level as antora yml file.

Under the modules folder we might have more than one module. But the ROOT module is required and this reserved directory name must be in all uppercase.

Other reserved directory names are "pages" and "partials" which we will write our content in.

And our navigation files that contains page references and will help us to navigate between pages.

All content outside of the modules folder won't be processed.
